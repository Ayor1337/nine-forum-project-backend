# 任务完成检查清单

完成编码任务后,应按以下步骤进行检查和验证:

## 1. 代码质量检查

### 编译检查
```bash
mvn clean compile
```
- 确保没有编译错误
- 检查警告信息并酌情处理

### 代码审查
- [ ] 代码遵循项目的命名约定
- [ ] 使用了适当的 Lombok 注解减少样板代码
- [ ] 分层架构正确 (Controller → Service → Mapper)
- [ ] 依赖注入使用正确
- [ ] 添加了必要的注释和文档

## 2. 功能测试

### 单元测试
```bash
mvn test
```
- [ ] 所有测试通过
- [ ] 新增功能有对应的测试用例
- [ ] 测试覆盖核心业务逻辑

### 集成测试
- [ ] 相关模块间的集成正常
- [ ] API 端点响应正确
- [ ] 数据库操作正确执行

### 手动测试
- [ ] 启动应用验证功能
```bash
cd web/web-app
mvn spring-boot:run
```
- [ ] 通过 Postman 或前端测试 API
- [ ] 验证 WebSocket 连接 (如适用)
- [ ] 检查日志无异常错误

## 3. 安全检查

- [ ] 敏感信息未硬编码 (使用环境变量)
- [ ] SQL 注入防护 (使用参数化查询)
- [ ] XSS 防护 (输入验证和输出转义)
- [ ] 认证和授权正确配置
- [ ] JWT 令牌处理安全

## 4. 性能检查

- [ ] 数据库查询优化 (避免 N+1 查询)
- [ ] 适当使用 Redis 缓存
- [ ] 避免不必要的对象创建
- [ ] 异步处理耗时操作 (RabbitMQ)

## 5. 配置检查

### 配置文件
- [ ] `application.yml` 配置正确
- [ ] 环境变量正确设置
- [ ] 端口配置无冲突

### 外部依赖
- [ ] MySQL 连接正常
- [ ] Redis 服务可用
- [ ] RabbitMQ 队列配置正确
- [ ] MinIO 文件上传正常
- [ ] Elasticsearch 索引正常 (如使用)

## 6. 文档更新

- [ ] 更新相关的 API 文档
- [ ] 更新 CLAUDE.md (如有架构变更)
- [ ] 添加必要的代码注释
- [ ] 更新 README.md (如有重大变更)

## 7. Git 提交

### 提交前检查
- [ ] 代码已格式化
- [ ] 移除调试代码和日志
- [ ] 删除未使用的导入
- [ ] 检查 git status 确认提交内容

### 提交
```bash
git status
git add .
git commit -m "描述性的提交信息"
```

### 提交信息规范
使用清晰的提交信息,例如:
- `feat: 添加用户头像上传功能`
- `fix: 修复聊天消息未读计数错误`
- `refactor: 重构权限验证逻辑`
- `docs: 更新 API 文档`
- `test: 添加账户服务单元测试`

## 8. 代码审查 (如适用)

- [ ] 创建 Pull Request
- [ ] 请求团队成员审查
- [ ] 处理审查意见
- [ ] 合并到目标分支

## 9. 部署准备 (生产环境)

- [ ] 更新生产环境配置
- [ ] 数据库迁移脚本准备
- [ ] 回滚计划准备
- [ ] 性能测试通过
- [ ] 安全扫描通过

## 最小化检查 (快速开发)

如果是快速开发/原型阶段,至少完成:
1. ✅ 编译通过 (`mvn clean compile`)
2. ✅ 基本功能测试通过
3. ✅ 无明显错误和异常
4. ✅ Git 提交有意义的信息
